Chapter 11 Further Functions

  • Function Properties and methods: All functions have a length property that returns the number of parameters it has.
  • Call() method can set a value of "this" inside a function to an object that is provided as first argument.
  • Memoization provides result caching. You can save it in a cache property to be used again later.
  • IIFE: Immediately Invoked Function Expression (invoked as soon as it's defined). Do this by placing parentheses at the end of the function definition. Useful to keep variable names wrapped in the scope of a function and not on the global level
  • No way to remove a variable from scope once it's declared
  • Functions define and redefine themselves. Assign an anonymous function to a variable that has the same name as the function
  • Init-time branching--optimizes to the browser being used
  • Recursive Functions: invokes itself until a condition is met.
  • callback hell... yikes! Too many nested callbacks making a pyramid of confusion.
  • Promises: represent future result of asynchronous operation. Don't do anything tha can't already be achieved with callbacks, but they simplify the process
  • promises are created using constructor function (resolve, reject)

Chapter 13 AJAX

  • Clients and Servers : JavaScript is clinet-side scripting language. Server side is required when requesting resources using Ajax. This involves server-side languages like PHP, Node.js, Ruby, etc.
  • CORS: Cross-origin resource sharing: A solution to the problem as it allows resources to be requested from another website outside the original domain.
  • AJAX: Asynchronous JavaScript and XML. Most common type of data returned is JSON, more lightweight and easier to parse than XML.
  • APIs: Application Programming Interface--a collection of methods that allow external access to another program or service. Many websites allow controlled access to data via public APIs. This is like the API we accessed in 230 for the weather site and weather data.
  • Fetch API: uses promises to avoid callback hell. Streamlines many concepts.
  • Redirect() method can redirect to another url.
  • text() method takes stream of text from response, reads it, and returns a promise
