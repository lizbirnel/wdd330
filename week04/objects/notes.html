Chapter 8 Forms

  • Search Form
  • See Search form console linked above for examples of code and how to access various elements of forms.
  • Accessing Form Elements: can use numerical index, name atribute, square bracket notation, etc to access the form object. We can access form controls using their 'name' attributes as if it were a property of the form object itself, and using square bracket notation as well.
  • Form Properties and Methods: form.submit() will submit a form automatically. Forms can be submitted manually by user clicking a button or input element if ithas the type attribute of submit, or even image. (type='submit' value='Submit', or type='image')
  • form.reset() method will reset all the form controls back to original initial values in HTML. If a button has reset for its type attribute, you don't need extra scripting to accomplish this. (type='reset'). Reset buttons are too easy to click and wipe out all data that's been entered, so they're not recommended and should be considered carefully before use.
  • form.action property can be used to set the action attribute. This can send it to a different URL to be processed on the server. (form.action = '(some location)')
  • Submitting Forms- This usually sends the content of the form to the server to be processes. JavaSCript can be used to intercept the form if you add an event listener.
  • Retrieving and Changing values from a form: Use a Value property to retrieve the text inside the field. You can use this to report back what was searched for. You can also change the value by using JS. (input.value = 'Search Here';). This leaves the text inside the input field box, though, and it needs to be deleted before entering text of your own as the user. Use focus and blur event handlers to fix that, and the default text can disappear when the input field box is clicked. This is similar to the placeholder attribute, but the placeholder attribute does not add an actual value to the input field.
  • Form Controls include input fields (text, passwords, check boxes, radio buttons, and file uploads), select menus (drop-down lists of options), text area elements (allows for longer text entry), and button elements to submit forms.
  • Hero Form Activity
  • Input fields are the most common types of form controls. There are several categories of input fields. Text input fields are for short pieces of text like usernames. type="text" is the default value, so it's not required, but it is best to use it anyway for the cohesivity of the code. Passord type for input fields works like a text type, but it conceals the characters as they are entered so they can't be read. Checkbox Input fields are used to select different options that can be true or false. The user can select more than one checkbox. Checkboxes have a checked property that tells if they have been checked or not with a boolean true or false answer. Radio Buttons are another type, but they can only have one option be selected. You can selct one checked radio button initially by setting a value="" to be checked in the html tag.
  • Hidden Input Fields can be created and are not displayed by the browser. They do have a 'value' attribute that can contain info submitted with the form. These are used to send info like settings, but shouldn't be used for sensitive data! It's easily visible in the html. File Input fields are used to upload files. Most browsers provide a browse button to search for a file. Other input types include number, tel, and color.
  • Select drop-down lists can be used for selecting one or more options from a list. Must use 'multiple' attribute to select more than one.
  • Text area elements are used for long pieces of text over multiple lines like a comment or blog post. They work like input fields, and access them using a 'name' attribute.
  • Buttons- default type is 'submit'. Another type is 'reset'. The other type is 'button'. This one just creates a clickable button that can have an event listener attached to it.
  • Form validation checks whether a user has entered the info in a form correctly. Can include a required field is completed, email address are valid format, number is entered instead of letter, password is minimum number of characters, etc. Validation can be on server side or client side using JavaSCript. It's best to use both client and server side validation. It's easy to bypass the JavaSCript on the user end and bypass the rules. JavaSCriptvalidation should be used to enhance UX when filling in a form by giving feedback about errors before it's submitted. Back it with more validation on the server side after submitting.

Chapter 12 - OOP

  • Object Oriented Programming is used to model representations of objects in the real world.
  • Encapsulation: inner workings are kept hidden inside object and only essential functionalities are exposed to end user, like the on button. Keeps all programming logic inside an object and makes methods available to implement functionality.
  • Polymorhpism: the same process can be used for differet objects. This means different objecs can share the same method but can override share methods with more specifics.
  • Inheritance: taking the features of one object and adding some new features. We can take something that already exists and inherit all properties and methods, but improve on it by adding properties and methods.
  • Classes: classes are used to define a blueprint for an object. Objectsa are created as instances of that class, and inherit the properites and methods fo that class.
  • Constructor Functions: functions that define properties and methods of an object.
  • JavaScript has built-in constructor functions for various things like Objects, Arrays, and Functions. It's easiest to just create them using the literal syntax, though. (const literalObject={}).
  • Class Declarations: You can use the new class declaration syntax that does the exact same thing as a constructor function, but it looks like writing a class in class-based programming languages.
  • All objects have constructor property that returns the function that created it.
  • Static keyword can be used to create static method. Static methods can be called by class directly rather than by instances of the class.
  • You can augment a class after it has been created by using the prototype property of the class. Use this to add properties and methods to already existing classes.
  • Overwrite prototype properties by assigning a new value to them. These prototypes can be used to add any new properties and methods after the class has been declared.

Chapter 15 - Modular JavaScript

  • Library is a piece of code that provides several methods to make it easy to achieve common tasks. Using a library can greatly reduce the amount of code needed to accomplish the same task.
  • jQuery is the most popular library used today. It's used in many commercial websites and has plugin system to make it easy to build common web page elements. It was released in 2006 as a DOM maniplation library. It's supported in older browsers like IE.
  • Underscore & Lodash are similar libraries of functions that provide functionalities. Both libraries are very similar. Lodash started as a fork of Underscore, but has diverged a bit. Underlying code is different. Lodash is thought of a "superset" of Underscore.
  • Advantages of libraries: tested by many users. Most likely ben optimized and battle-tested. You can be confident that code is well-built for many browsers.
  • Disadvantages of libraries: Need to include the code for the library as well as your own code. Increases amount of code downloaded by a website and increases page weight and performance issues. Libraries might fail to implement the functionality in a precise way you want it to perform. You might have to write your own functions to achieve what you are looking for. Code can run slower when using a library because of more lines of code.
  • Modular JavaSCript is self-contained piece of code that provides functions and methods that can be used in other files and other modules. Helps organize code in separate reusable files. You can easily swap one module for another without affecting other parts of a project.
