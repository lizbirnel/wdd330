Chapter 8 CSS Transforms and Transitions

Transform Property

- lets you lets you translate, rotate, scale, and/or skew any element on the page.

- can move elements left, right, up, down on the page

Translation

-Translate functions allow you to move elements left, right, up or down.

-Similar to postion: relative;

-When you use translation function, it doesn't impact the flow of the document

-Translated elemnts can only be moved relative to current position.

-moves along x and y axes (+ moves up and right, - moves down and left)

Scaling

-scales an element by defined factors horizontally then vertically.

-if only one value is provided, it counts for both x and y values (maintaining aspect ratio)

-scale(1) leaves same size, scale(2) doubles, scale(0.5) halves

-scaled elements grow outwards or scale inwards from the center. Center stays the same. This can be changed with Transform-origin property

-declare scale in the same transform: line as translate, just separate by a space

-scaling doesn't impact document flow. If you scale inline-block elements, text around it will not adjust for the changes and could cause problems. Transforming does not cause reflow of all elements.

Rotation

-rotates element around point of origin by specified angle degree value.

-default, the point of origin is center of element.

-positive degrees move clockwise, negative move counterclockwise

Skew

-specifies a skew of x and y axes: skew(x,y)

-if seconded parameter is left off, skew only applies to x axis.

-can also skew individual axes with skewX() and skewY()

Changing Origin of Transform

-you can control origin for transforms by using transform-origin property

Same syntax as background-position, defaults to center of object

Choose Ordering Carefully

-order of transform functions matters— if you rotate before translating, your translate direction is on the rotated axis.

Transitions

-transitions require less client-side processing than animations and can appear smoother

-allow values of CSS properties to change over time

-can create gradual changes instead of sudden, like a hover that changes link colors slowly

-help create more dynamic pages with subtle animations

-declared along with regular styles on an element

Steps for simple transition in CSS:

  1. Declare original state of element in default style declaration.
  2. Declare final state of transitioned element (for example, :hover state)
  3. Include transition functions in your default style declaration using transition properties, including: transition-property, transition-duration, transition-timing-function, and transition-delay.

Transition-duration Property

-sets how long transition will take to go from default state to the transitioned state.

-can be specified in seconds (s) or milliseconds (ms).

-200ms (or .2 s) is generally considered optimum time for a transition. If slower, the website seems slow, faster might be too subtle.

transition-timing-function Property

-lets you control the pace of transition in more detail.

-common timing-functions are ease, linear, ease-in, ease-out, or ease-in-out

-you can even describe your own cubic-bezier function with four numeric parameters, like: cubic-bezier(0,0,1,1).

transition-delay Property

-you can introduce a delay before the transition begins.

-default si 0, but add a time in ms or s to delay.

transition Shorthand Property

-shorten your lines of code for a single transition by using shorthand

-separate propertiesof transitions by spaces, can be in any order If you use a delay you must include a duration, and duration comes before delay in order.

Multiple Transitions

-transitions allow for multiple in one call, like changing color at same time as changing rotation and size

-you can shorthand to specify multiple transitions at different durations, delays, and timing functions. Specify all transition values for each transition, separated by a comma between transitions.
For example— transition: transform 0.2s ease-out-50ms, color 0.1s linear 50ms;

Animations

-allow you to control each step fo an animation via keyframes

-a keyframe is a snapshot that defines a starting or end point of any smooth transition

-transitions limit you to a start and end keyframe, but animations allow us to add any number of keyframes in between to make animations more complex.
